# Features - Veil Shine

This document provides a comprehensive list of features for the Veil Shine platform, categorized by their priority (P0, P1, P2) and offering a brief description.

## 1. Feature Prioritization Legend

*   **P0 (Must-Have)**: Essential for the core functionality and initial launch of the platform. Without these, the product is not viable.
*   **P1 (Should-Have)**: Important features that significantly enhance the user experience and value proposition, but are not strictly necessary for the MVP. These will be included in subsequent releases post-MVP.
*   **P2 (Could-Have)**: Desirable features that add polish, convenience, or cater to niche use cases. These will be considered for later stages of development, subject to resources and user feedback.

## 2. Feature List

### 2.1. Core Anonymity & Privacy

| Priority | Feature Name | Description |
|:---------|:-------------|:------------|
| P0       | Anonymous Account Creation | Users can create an account using only a self-chosen handle, without providing any PII (email, phone, real name). |
| P0       | Architectural Anonymity | System design ensures that user activity cannot be linked to real-world identities, even by platform administrators. |
| P0       | Client-Side Passphrase Recovery | Optional, user-generated and client-stored passphrase for account recovery. Server never stores or transmits this passphrase. |
| P0       | Basic Anonymous Profile | A simple profile showing only the chosen handle and a system-generated or selected anonymous avatar. |
| P1       | End-to-End Encrypted Direct Messages | Secure, private one-on-one communication where only sender and receiver can read messages. |
| P1       | AI-Modified Media Labeling | All images or audio generated/modified by AI are clearly and immutably labeled as such. |
| P2       | Zero-Knowledge Proofs for Actions | Implement advanced cryptographic techniques to verify user actions without revealing any additional information about the user. |

### 2.2. Content Creation & Sharing

| Priority | Feature Name | Description |
|:---------|:-------------|:------------|
| P0       | Text-Based Anonymous Posts | Users can publish anonymous text posts (confessions, stories, questions) up to 500 characters. |
| P0       | Post Categorization | Users can assign their posts to predefined categories for better discoverability. |
| P0       | Anonymous Comments & Replies | Users can comment anonymously on posts and other comments, supporting threaded discussions. |
| P0       | Anonymous Reactions | Users can react to posts and comments using a predefined set of anonymous emojis/reactions. |
| P1       | AI-Generated Image/Audio Attachments | Users can attach AI-generated images or short audio snippets to their posts, with AI modification clearly indicated. |
| P1       | Draft Saving | Users can save incomplete posts as drafts for later editing and publishing. |
| P2       | Scheduled Posts | Users can schedule posts to be published at a future date and time. |
| P2       | Polls/Questions in Posts | Ability to embed anonymous polls or multi-choice questions within posts. |

### 2.3. Content Discovery & Feed

| P1       | Advanced Search & Filters | Comprehensive search functionality with filters for sentiment, category, time, and popularity. |
