# Minimum Viable Product (MVP) - Veil Shine

## 1. Introduction

This document outlines the Minimum Viable Product (MVP) for Veil Shine, focusing on the core features necessary to launch a functional, valuable, and anonymous social platform. The MVP aims to validate key assumptions about user engagement with anonymous content and the effectiveness of AI-driven moderation, while establishing a strong foundation for future development.

## 2. MVP Goals

*   **Validate Core Anonymity Proposition**: Ensure users can genuinely post and interact without fear of identity exposure.
*   **Establish Basic Content Loop**: Enable users to create, discover, and react to anonymous content.
*   **Prove AI Moderation Effectiveness**: Demonstrate the AI's ability to maintain a safe environment with minimal human intervention.
*   **Gather Initial User Feedback**: Collect data on user behavior, preferences, and pain points to inform future iterations.

## 3. In-Scope Features (MVP)

### 3.1. User Onboarding & Anonymity
*   **Anonymous Account Creation**: Users can create an account with a unique, self-chosen handle. No email, phone number, or real name required. A unique, non-identifiable user ID is generated internally.
*   **Client-Side Passphrase Recovery**: Optional, client-side generated and stored passphrase for account recovery. Server never sees or stores this passphrase.
*   **Basic Profile**: A simple, anonymous profile consisting only of the chosen handle and a system-generated avatar (or a selection from predefined anonymous avatars).

### 3.2. Content Creation
